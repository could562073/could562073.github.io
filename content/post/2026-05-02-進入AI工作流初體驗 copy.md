一個工程師進入AI agent工作流初體驗(一)
===


## 開始學習
最近開始正式的研究AI Agent coding flow 的工作流，沒碰還好一碰就真的驚為天人，沒想到在我沒寫程式這短短的6~7個月，AI的進化程度已經遠遠超出我的想像.... 當然也有一些我看到AI目前的盲點

那我基本上是從一些youtube的介紹跟文章開始看，一步一步先把claude code的環境建置起來，因為這個agent 只能run 在linux，所以必須來搞一下windows 11的 WSL，還順便建好了docker、起了一個postgres容器，連動到navicat方便之後開發專案使用

(題外話: windows 的 WSL 跑linux強制綁hyper-v 導致ThrottleStop降壓軟體就沒辦法變動硬體，這點非常困擾我，只能切兩個windows配置，一個關閉hyper-v，一個原配置，每次開機做選擇)

## 過程紀錄

基本上我就是開始研究claude code，最近想幫女朋友的媽媽處理紙本記帳的一些問題，且順便把程式設計的腦袋找回來，所以我開始設計APP做店家的每日記帳小APP，非常適合當作第一個AI專案，然後就在gitHub上建了一個專案，然後用claude 梳理了一個初版的Prompt，在vs code 與 Claude code做一頓輸出之後，整個AI實作過程大概一個小時半左右，完成初版專案


### 發想階段
在專案發想階段，我有明確的說我要先做一個MVP，所以提出了幾個方向跟發散想法，然後收斂

這些都花費不到幾分鐘，因為方向其實算很明確，功能也非常小，所以馬上就完成初期的定案

![image](https://hackmd.io/_uploads/HJypnKVRbe.png)

![image](https://hackmd.io/_uploads/SkrbaYNRWe.png)


當然其實這個記帳系統我自己的想法是直接用Google Sheet來做DB，好處有很多

1. 因為都是純文字，所以不需要什麼複雜的DB結構
2. 存在Google Drive等於資料永遠在，什麼時候想去總結都行(Google萬歲)
3. 之後要做excel報表分析，可以直接在上面做，甚至未來需要把此系統擴展成可視覺化的圖表資料，都非常方便
4. 後端成本 = 0 (這才是最重要的!)，但未來如果要擴展更多複雜功能可能就真的還是需要服務器、DB等等的內容了，不過現階段完全不需要

### Claude 會解釋為什麼做這樣的design

我覺得都合理，但是不是符合真正使用者需求，就是需要我們去把關，以免被唬得一愣一愣的

像是這個APP AI考慮了使用者會有多個，但實際上記帳這件事可能只會有老闆或是店長一個人負責做，所以整體架構可能不需要這麼大，甚至前端就能完成，但考慮到資料持久性還是需要有個地方儲存這些數據避免後續資料遺失的問題


![image](https://hackmd.io/_uploads/rJMCAKVAWe.png)


### 盲點

但我發現AI的盲點就是，因為現在的AI有Memory功能，所以在討論的時候他會把之前你說過的東西考慮進來，然後我昨天也太興奮在玩這個AI

講到Tech Stack的時候 Backend這塊是完全不需要的，直接對google api就能實現類似後端的功能
