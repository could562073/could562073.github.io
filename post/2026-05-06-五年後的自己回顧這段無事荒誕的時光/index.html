五年後的自己，回顧這段無事荒誕的時光

Burnout

人生35，27歲決定去澳洲打工度假2年，那是我人生最開心的一段時光，也是讓我變得更獨立勇敢，回台灣後還是回到系統廠上班，但後來覺得一成不變的工作不適合我，29歲決定去資策會從頭開始去學習軟體，從硬體工程轉到軟體工程，那時候的我，充滿幹勁，覺得自己做的一個很棒的決定，到現在依然沒有後悔當時做的決定

記得那時候給自己五年時間，看看現在五年後的我 五年後我已經是公司核心且資深的工程師\組長，然後在即將邁入第六年的時候，我決定離職

工作五年多後發現自己整個身心靈都burnout，決定給自己一段時間思考，未來的方向和修復沉澱自己

焦慮

離職後其實是在準備雅思，想強化自己的競爭力，但一兩個月後就開始對各種事物充滿焦慮，不管是未來、自己、英文等等大小事情，可能真的身心俱疲了，所以真的對於各種事物充滿焦慮以及對自己開始沒自信，對於原本熱愛寫程式、解決問題的我，完全不想在寫任何一行程式碼，也不想做任何事情，提升英文這件事上也開始變得零零散散的準備

離開

就這樣陸陸續續地，期間還是偶爾會認真念英文一段時間，但之後就會完全不想念，養養魚，種仙人掌、養花養草，後來我們養的小鬥魚去當天使了，是跟女朋友一起養的，她哭得很慘，因為這是她第一次擁有自己的寵物，所以特別難過

雅思

後來就開始認真準備雅思的英文考試，其實本來預計7月準備到9月然後考雅思，但計畫總是趕不上變化，後來經歷一些事情後一直往後延，2026年的一月底才終於考完，把讀英文這件事情做個總結，有了全心認真準備過雅思這段經歷，讓我英文的聽說讀寫都比以往更加順利，至少現在再聽一些英文內容時的敏銳度已經大幅的進步，還有閱讀英文文章的能力，寫作跟口說就達到堪用的程度，寫作真的很困難，沒有在生活上有一些英文的積累真的幾乎速成不了，且以前作文也沒有寫得多好多棒

回到職場的準備

半年完全沒寫任何一行程式碼的我，還是不知道該做些什麼，雖然一直有在關注科技事務和時事新聞，但事實上到底對於軟體工程師影響多大，我並不清楚，當時只覺得感覺軟體工程師好像受影響很大，但是不知道確切影響範圍有多深多廣，當時還在職時，也有用AI輔助開發，auto complete在我那時候是最夯的，可以補完剩下的程式碼已經是可以加速整個開發速度，且常使用chatGPT來排查一些latency code內容很多的東西，一段一段解析然後再做判斷去解決問題

現在

面對許久沒回到人間的我，開始調整自己的各種狀態，開始研究vibe coding、AI、現在的新科技、機器人、coding現在如何了等等，發現僅僅半年，這些事物進步的異常快速，AI的演進已經快到我追不上，所以我後來決定跳到最新的 agent事物來做第一線的理解

透過理解claude code，開始付費使用來開發友人想要的店家線上記帳系統，才發現驚為天人，單單下指令就能開發出足夠完整的application，且串接第三方的東西都是幾分鐘就能完成的事情，以前要串一個google third party login，就差不多要幾天的時間研究，怎麼整合進原有的系統架構，現在串一個google sheet不需要2分鐘，且我直接當成Database來用，省下了後端Server維護的成本 ( 抱歉了google，我就是個卑鄙的免費仔 )

再用agent開發後我開始思考，難道技術真的不再重要了嗎?

以前持續累計的實戰經驗、需求設計、程式碼細節、solid、clean code、concurrency，這些還有任何實質意義嗎?

但是後來想想，可能還是很重要，尤其是越大、越精、越複雜的系統之中，更需要這些實務或是具體碰過的事件才能在某些問題發生之前意識到，或是預判，AI當然也能做得更好，但是在越複雜的場景，需要具備夠多的經驗或是足夠敏感的工程師，才能具備判斷的能力，現在的AI還不足以掌握太過複雜的平台或系統所有的細節跟設定

這就像是 AI 能幫你做一個簡單的系統，但是要開始擴展時，整體的架構可能都要打亂，且可能還要在不影響線上系統還要做原本的資料整合種種情況之下，可能會花更多時間在處理重構或是改造的問題上，取捨怎麼做、潛在問題會是什麼，安全性問題如何、有沒有被其他agent 攻擊或竊取資訊的可能性 ，這些經驗在大部分一般人去做這件事，可能都無可避免

我在開發線上記帳系統時就有發現，如果AI沒辦法理解你想做的事物，或是你自身的認知不足以描述的足夠清楚的情況之下，常常會來來回回的改，尤其是較大的改動，很常改A壞B，但也可能跟使用的模型有關，我只用sonnet4.6做開發，沒用到opus4.7( token too expensive) -2026/05/19的claude code

所以必須說AI的進步還是非常神速的，但我覺得工程師的價值不是體現在是否熟練於技術，而是經年累月出的那種涵養、敏銳、基礎知識、問題解決、效率這些是非工程師沒有的特殊能力，而這些能力，正是AI無法取代的
