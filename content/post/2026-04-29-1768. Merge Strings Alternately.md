### 主要用了什麼方法：
For迴圈

### 用了多久: 
將近兩個小時

### 卡在哪裡：

因為用了for迴圈需要先定義長度，所以卡很久，剩餘元素一直處理不好，卡在兩個長度不一致時會有的情況，一直在想怎麼處理剩餘元素撈取的問題

#### Time Complexity:  
**O(n)**

#### Space Complexity:  
**O(m+n)**

## My Solution:

```java
class Solution {
    public String mergeAlternately(String word1, String word2) {
        StringBuilder sb1 = new StringBuilder();
        int maxlength;
        if (word1.length() >= word2.length()) {
            maxlength = word1.length();
        } else {
            maxlength = word2.length();
        }

        for (int i = 0; i < maxlength; i++) {
            if (i < word1.length()) {
                sb1.append(word1.charAt(i));
                if (i < word2.length()) {
                    sb1.append(word2.charAt(i));
                }
            }
            if (i == word1.length()) {
                for (int k = i; k < word2.length(); k++) {
                    sb1.append(word2.charAt(k));
                }
            }
        }
        return sb1.toString();
    }
}

```

## Best Practice (from claude AI)

```java
class Solution {
    public String mergeAlternately(String word1, String word2) {
        StringBuilder sb = new StringBuilder();
        int i = 0;
        while (i < word1.length() || i < word2.length()) {
            if (i < word1.length()) sb.append(word1.charAt(i));
            if (i < word2.length()) sb.append(word2.charAt(i));
            i++;
        }
        return sb.toString();
    }
}
```

### 學到什麼：

與其先算邊界，不如讓條件自己管邊界，claude的最佳實踐用的是while 迴圈，直接判斷兩個長度即可疊代所有元素出來
